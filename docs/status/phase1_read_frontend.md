# Phase 1 read frontend

This phase formalizes the WhatsHap-style read processing stack as a first-class AWPhase frontend.

## Package
- `python/awphase_py/read_frontend/run_frontend.py`
- `python/awphase_py/read_frontend/write_branch_configs.py`

## Outputs per run
- readobs v3
- selected reads
- tagged reads
- superreads
- selected/tagged/superreads JSON read sets

## Purpose
Make the read front end stable and reusable before scaffold/panel fusion.
